# Phase 1 测试指南

## 1. 安装依赖

在 VS Code 终端或 PowerShell 中运行：

```bash
cd D:\111\jimi-ai-study-main
npm install react-markdown react-syntax-highlighter
```

## 2. 启动开发服务器

```bash
npm run dev
```

打开浏览器访问 `http://localhost:3000`

---

## 3. 测试清单

### ✅ 3.1 聊天页面正常加载

1. 点击首页「进入学习」或「开始解题」
2. 确认 `/problems` 页面正常加载
3. 确认左侧栏显示「今日/昨日/更早」会话分组
4. 确认可以新建对话、切换会话、删除会话

### ✅ 3.2 Markdown + 代码高亮

1. 在聊天框输入一段代码，例如：
   ```
   请帮我分析这段代码的问题：
   ```cpp
   int main() {
       int a[10];
       cout << a[10];  // 这里有问题吗？
       return 0;
   }
   ```
   ```
2. 发送后观察 AI 回复：
   - 代码块是否有深色背景 + 语法高亮？
   - 代码块右上角是否有「复制」按钮？
   - 点击复制后是否显示「已复制」？
3. 测试 Markdown 格式：
   - 输入「请用列表说明数组越界的危害」
   - 确认 AI 回复中的列表、加粗、标题等格式正确渲染

### ✅ 3.3 流式响应

1. 发送一个需要较长回复的问题，例如：
   ```
   请详细解释快速排序算法，包括代码实现和复杂度分析
   ```
2. 观察 AI 回复是否**逐字出现**（像打字机效果）
3. 确认回复过程中「基」头像旁有 3 点动画
4. 确认回复完成后动画消失

### ✅ 3.4 模式持久化

1. 在聊天页面底部的下拉框切换模式（如从「C++ 基础」切换到「通用逻辑」）
2. 刷新浏览器页面（F5）
3. 确认下拉框仍然显示上次选择的模式

### ✅ 3.5 每日一题流程

1. 回到首页 `/`
2. 点击「开始解题」
3. 确认自动跳转到 `/problems` 并自动发送题目
4. 确认 AI 开始回复解题思路

### ✅ 3.6 边界情况

1. **空消息**：不输入内容直接点击发送，确认无反应
2. **快速发送**：连续快速点击发送，确认不会重复发送
3. **网络中断**：断开网络后发送消息，确认显示「基米现在有点忙」
4. **长代码**：粘贴 50 行以上代码，确认代码块可滚动、可复制

## 4. 常见问题排查

| 问题 | 解决方案 |
|---|---|
| `npm install` 报错 | 检查 Node.js 版本（需 ≥ 18），或尝试 `npm install --legacy-peer-deps` |
| 页面白屏 | 检查浏览器控制台（F12 → Console）是否有红色报错 |
| AI 不回复 | 检查 `.env.local` 中 `OPENAI_API_KEY` 是否配置 |
| 代码块无高亮 | 确认 `react-syntax-highlighter` 已正确安装 |
| 流式响应卡顿 | 正常现象，首次加载模型需要 2-5 秒 |
